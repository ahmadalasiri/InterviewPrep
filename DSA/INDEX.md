### Beginner (4-6 Weeks)

**Week 1-2: Foundations**

- Arrays & Strings (Two Pointers, Sliding Window)
- Basic array manipulations
- Simple string problems
- Practice: 20-30 easy problems

**Week 3-4: Linear Data Structures**

- Linked Lists (traversal, reversal)
- Stacks & Queues (basic operations)
- Practice: 20-30 easy-medium problems

**Week 5-6: Trees & Hash Tables**

- Binary Trees (DFS, BFS)
- Hash Table operations
- Practice: 20-30 medium problems

### Intermediate (6-8 Weeks)

**Week 1-2: Advanced Trees & Graphs**

- BST operations
- Graph traversals
- Basic shortest path
- Practice: 30-40 medium problems

**Week 3-4: Dynamic Programming**

- 1D DP (Fibonacci, Climbing Stairs)
- 2D DP (LCS, Edit Distance)
- Practice: 30-40 medium problems

**Week 5-6: Backtracking & Greedy**

- Subsets, Permutations
- Interval problems
- Job scheduling
- Practice: 20-30 medium problems

**Week 7-8: Advanced Topics**

- Heaps & Priority Queues
- Advanced DP
- Bit Manipulation
- Practice: 20-30 medium-hard problems

**Week 1-2: Hard DP & Graphs**

- Advanced DP patterns
- Dijkstra, Bellman-Ford

- Practice: 30-40 hard problems

**Week 3-4: Advanced Algorithms**

- Union-Find
- Trie

- Practice: 20-30 hard problems

**Week 5-6: System Design & Mock Interviews**

- Design problems
- Mock interviews
- Review weak areas
- Practice: 20-30 mixed difficulty

## üéì Company-Specific Focus

### FAANG (Meta, Amazon, Apple, Netflix, Google)

- **Focus**: Medium to Hard problems
- **Topics**: Trees, Graphs, DP, System Design
- **Practice**: 150-200 problems minimum
- **Time**: 3-6 months preparation

- **Focus**: Medium problems, code quality
- **Topics**: Arrays, Strings, Trees, DP
- **Practice**: 100-150 problems
- **Time**: 2-4 months preparation

### Startups

- **Focus**: Practical problems, implementation
- **Topics**: Arrays, Hash Tables, Basic DS
- **Practice**: 75-100 problems
- **Time**: 1-3 months preparation

## üìû Getting Help

### Stuck on a Problem?

1. Read problem statement carefully
2. Try brute force first
3. Look for patterns in examples
4. Check the hints
5. Study the solution approach (not code)
6. Implement yourself
7. Review optimal solution

### Need Clarification?

- Check problem discussion section
- Watch solution videos
- Join study groups
- Ask on Stack Overflow
