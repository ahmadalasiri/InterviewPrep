## üéì Interview Strategy

### Before the Interview

1. **Review Fundamentals**: Focus on complexity analysis and common patterns
2. **Practice Communication**: Explain solutions out loud
3. **Set Up Environment**: Test your coding environment beforehand
4. **Rest Well**: Get good sleep the night before

### During the Interview

1. **Understand the Problem**

   - Ask clarifying questions
   - Confirm inputs, outputs, constraints
   - Work through 1-2 examples

2. **Plan Your Approach**

   - Discuss brute force first
   - Identify optimization opportunities
   - Explain time/space complexity

3. **Write Code**

   - Use meaningful variable names
   - Write clean, readable code
   - Think out loud

4. **Test Your Solution**

   - Walk through with example
   - Check edge cases
   - Identify potential bugs

5. **Optimize if Needed**
   - Discuss trade-offs
   - Implement improvements

### Common Mistakes to Avoid

- ‚ùå Jumping into code without understanding
- ‚ùå Not asking clarifying questions
- ‚ùå Ignoring edge cases
- ‚ùå Poor variable naming
- ‚ùå Not explaining thought process
- ‚ùå Getting defensive about mistakes
- ‚ùå Not testing the code

## üí° Key Concepts to Master

### Pattern Recognition

- **"Find pair/triplet"** ‚Üí Two Pointers or Hash Map
- **"Substring/subarray"** ‚Üí Sliding Window
- **"Connected components"** ‚Üí DFS/BFS or Union Find
- **"Shortest path"** ‚Üí BFS (unweighted) or Dijkstra (weighted)
- **"Optimize/count ways"** ‚Üí Dynamic Programming
- **"Generate all possibilities"** ‚Üí Backtracking
- **"Greedy choice"** ‚Üí Greedy Algorithm

### Complexity Intuition

- Nested loops ‚Üí O(n¬≤) or worse
- Single loop ‚Üí O(n)
- Divide in half ‚Üí O(log n)
- Sorting ‚Üí O(n log n)
- All subsets ‚Üí O(2‚Åø)
- All permutations ‚Üí O(n!)

## üèÜ Company-Specific Tips

### FAANG/Big Tech

- Focus on medium to hard problems
- Expect 2-3 coding rounds
- System design for senior roles
- Strong emphasis on optimization
- **Practice**: LeetCode Top Interview Questions

### Startups

- More practical coding problems
- Often use real work scenarios
- May include take-home assignments
- Emphasis on clean, maintainable code
- **Practice**: Full-stack problems, API design

### Finance/Trading

- Focus on mathematics and optimization
- Low-latency, high-performance code
- Probability and statistics questions
- **Practice**: Math-heavy problems, C++
